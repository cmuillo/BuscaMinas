¿Qué es una Matriz?

Concepto clave

Dos tipos de recursión

Recursión por PILA

Después de la llamada recursiva queda una operación pendiente. El programa la guarda en la pila de llamadas (stack) y la ejecuta al regresar.


Señal en el código:

return fila[col] + f(fila, col+1)
# ↑ suma FUERA de la llamada

Recursión por COLA

La llamada recursiva es lo último que ocurre. El resultado parcial viaja como un parámetro extra llamado acumulador.


Señal en el código:

return f(fila, col+1, acum+fila[col])
# ↑ la llamada ES todo el return
Recursión por Pila

¿Cómo funciona la pila?

Cada llamada apila una operación pendiente. Las sumas se resuelven al regresar, de abajo hacia arriba.

Pila de llamadas para suma_fila([1,2])

col=2 → devuelve 0
col=1 → espera: 2 + ???
col=0 → espera: 1 + ???

▲ Se apilan al bajar · Se resuelven al subir

suma_fila([1,2], col=0)
 = 1 + suma_fila([1,2], col=1)
    = 2 + suma_fila([1,2], col=2)
        = 0 ← caso base
    ← 2 + 0 = 2
 ← 1 + 2 = 3
Por Pila

Código — Recursión por Pila

def suma_fila_pila(fila, col=0):
  if col == len(fila):
      return 0                       # caso base
  return fila[col] + suma_fila_pila(fila, col + 1)
  #      ↑────────── operación PENDIENTE en la pila

def suma_matriz_pila(matriz, fila=0):
  if fila == len(matriz):
      return 0                       # caso base
  return suma_fila_pila(matriz[fila]) + suma_matriz_pila(matriz, fila + 1)
  #      ↑──────────────────────────── operación PENDIENTE en la pila

  • El caso base devuelve 0 (neutro de la suma).
  • Las sumas se acumulan al volver de la recursión más profunda.
Recursión por Cola

¿Cómo funciona el acumulador?

El resultado parcial viaja hacia adelante como parámetro. Al llegar al caso base, el acumulador ya tiene el resultado final.

Progreso del acumulador para suma_fila([1,2])

col=0→
acum = 0
pasa 0+1=1
col=1→
acum = 1
pasa 1+2=3
col=2→
acum = 3 ✓
caso base

▶ El resultado se construye hacia adelante

suma_fila_cola([1,2], col=0, acum=0)
 → suma_fila_cola([1,2], col=1, acum=1)
   → suma_fila_cola([1,2], col=2, acum=3)
     → caso base: devuelve acum=3
   ← 3 (sin operaciones pendientes)
Por Cola

Código — Recursión por Cola

def suma_fila_cola(fila, col=0, acum=0):
  if col == len(fila):
      return acum                    # caso base: ya tenemos el resultado
  return suma_fila_cola(fila, col + 1, acum + fila[col])
  #       ↑────────────────────────────────────────────────
  #       la llamada es todo el return → nada pendiente después

def suma_matriz_cola(matriz, fila=0, acum=0):
  if fila == len(matriz):
      return acum                    # caso base
  return suma_matriz_cola(matriz, fila + 1, acum + suma_fila_cola(matriz[fila]))

  • El caso base devuelve el acumulador (ya tiene todo sumado).
  • La suma se hace antes de la llamada, en el argumento acum.
Comparación

Pila vs Cola — diferencias clave

Por PilaPor Cola
¿Dónde se acumula?En la pila de llamadas (al regresar)En el parámetro acum (al avanzar)
Caso base devuelve0 (neutro de suma)acum (resultado final)
Operaciones pendientesSí — una suma por nivelNo — nada después de la llamada

Ejercicio 1

Sumar una sola fila: [5, 10, 15]

Por PILA

f_pila([5,10,15], col=0)
 = 5 + f_pila(col=1)
   = 10 + f_pila(col=2)
     = 15 + f_pila(col=3)
        = 0 ← base
= 5+10+15+0 = 30

Por COLA

f_cola([5,10,15], col=0, acum=0)
 → col=1, acum=5
   → col=2, acum=15
     → col=3, acum=30
        → base: return 30
= 30

Resumen

Lo que aprendimos hoy

  • Una matriz en Python es una lista de listas.
  • Recursión por PILA: hay una operación después de la llamada → se guarda en la pila y se resuelve al regresar.
  • Recursión por COLA: la llamada es lo último → el resultado parcial viaja como parámetro (acumulador).
  • Ambas producen el mismo resultado; difieren en cómo gestionan la memoria.

Pregunta de reflexión

¿Podrías reescribir las funciones para encontrar el máximo de la matriz en lugar de la suma? ¿Qué cambiarías en el acumulador?

Respuesta

Máximo de la matriz — ¿qué cambia?

Solo cambian dos cosas: el valor inicial del acumulador y la operación que se aplica.

Por PILA

def max_fila_pila(fila, col=0):
  if col == len(fila) - 1:
      return fila[col]  # último elemento
  return max(fila[col], max_fila_pila(fila, col+1))
  # ↑ max() en vez de +

def max_matriz_pila(matriz, fila=0):
  if fila == len(matriz) - 1:
      return max_fila_pila(matriz[fila])
  return max(max_fila_pila(matriz[fila]),
             max_matriz_pila(matriz, fila+1))

Por COLA

def max_fila_cola(fila, col=0, acum=float('-inf')):
  if col == len(fila):
      return acum   # caso base
  return max_fila_cola(fila, col+1, max(acum, fila[col]))
  # ↑ max(acum, ...) en vez de acum + ...

def max_matriz_cola(matriz, fila=0, acum=float('-inf')):
  if fila == len(matriz):
      return acum
  return max_matriz_cola(matriz, fila+1,
    max(acum, max_fila_cola(matriz[fila])))

Suma → neutro es 0

Cualquier número + 0 = ese número.
Por eso acum empieza en 0.

Máximo → neutro es -∞

Cualquier número es mayor que -inf.
Por eso acum empieza en float('-inf').
