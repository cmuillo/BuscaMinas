🌳 Árboles Binarios — Ejercicios de Examen

¿Qué es un Árbol Binario?

🌳 Definición

Un árbol binario es una estructura donde cada nodo tiene como máximo 2 hijos: uno izquierdo y uno derecho. El nodo superior es la raíz.

🔗 Nodo

Cada nodo guarda un valor (dato) y dos referencias: izquierdo y derecho. Si no tiene hijo, la referencia vale None.

📏 Altura

Número de niveles del árbol. Un árbol de un solo nodo tiene altura 1. Se calcula con recursividad.

🌍 Usos reales

Sistemas de archivos, árboles genealógicos, inteligencia artificial (minimax), bases de datos, autocompletado, etc.

Anatomía de un árbol

class Nodo:
    def __init__(self, valor):
        self.valor     = valor  # dato que guarda el nodo
        self.izquierdo = None  # hijo izquierdo (vacío al inicio)
        self.derecho   = None  # hijo derecho   (vacío al inicio)

Ejercicio 1: Recorridos del Árbol

🎯 Problema

Dado un árbol genealógico, "leer" todos los nodos en tres órdenes distintos: inorden, preorden y postorden.

👨‍👩‍👧‍👦 Caso real

Un árbol de carpetas en Windows/Mac. Al listar archivos en orden alfabético se usa inorden; para copiar la estructura completa se usa preorden.

Código — Recorridos

# Árbol ya construido globalmente (raiz = Nodo("Ana"), ...)

# Inorden: izq → raíz → der
def inorden(nodo, res=None):
    if res is None: res = []
    if nodo is None: return res  # caso base
    inorden(nodo.izquierdo, res)
    res.append(nodo.valor)
    inorden(nodo.derecho, res)
    return res

# Preorden: raíz → izq → der
def preorden(nodo, res=None):
    if res is None: res = []
    if nodo is None: return res
    res.append(nodo.valor)
    preorden(nodo.izquierdo, res)
    preorden(nodo.derecho, res)
    return res

# Postorden: izq → der → raíz
def postorden(nodo, res=None):
    if res is None: res = []
    if nodo is None: return res
    postorden(nodo.izquierdo, res)
    postorden(nodo.derecho, res)
    res.append(nodo.valor)
    return res

# Altura: 1 + max(altura_izq, altura_der)
def altura(nodo):
    if nodo is None: return 0  # caso base
    return 1 + max(altura(nodo.izquierdo), altura(nodo.derecho))

🧠 Clave: Recursividad

Cada función se llama a sí misma con un subárbol más pequeño. El caso base (nodo is None) detiene la recursión.

🧠 ¿Cuándo usar cada uno?

  • Inorden → leer valores ordenados (BST)
  • Preorden → copiar/serializar árbol
  • Postorden → eliminar nodos, evaluar expresiones

📏 Altura recursiva

La altura es 1 + max(altura_izq, altura_der).
Compara los dos subárboles y queda con el más alto.

Ejercicio 2: Árbol BST — Agenda

🎯 Problema

Implementar una agenda de contactos usando un BST. Los nombres se guardan en orden alfabético de forma automática: menores a la izquierda, mayores a la derecha.

📱 Caso real

Los motores de bases de datos usan BSTs (índices B-Tree) para hacer búsquedas en millones de registros en milisegundos.

⚡ Ventaja

En un árbol equilibrado de N nodos, buscar tarda O(log N) pasos. Con 1 millón de contactos: solo ~20 comparaciones.

La regla del BST

Si nombre < nodo actual → ir a la IZQUIERDA
Si nombre > nodo actual → ir a la DERECHA
Si nombre == nodo actual → ¡ENCONTRADO!

BST resultante — insertar: Mario, Ana, Pedro, Carlos, Lucía, Sofía, Beatriz

Búsqueda de "Lucía" — paso a paso

Mario
¿"Lucía" < "Mario"? Sí → ir a la IZQUIERDA
Ana
¿"Lucía" < "Ana"? No → ir a la DERECHA
Carlos → Lucía
¿"Lucía" < "Carlos"? No → ir a la DERECHA → ¡Encontrado en 3 pasos!

Código — BST (Insertar y Buscar)

class AgendaBST:
    def __init__(self):
        self.raiz = None  # árbol vacío

    def insertar(self, nombre):
        if self.raiz is None: self.raiz = Nodo(nombre)
        else: self._insertar(self.raiz, nombre)

    def _insertar(self, nodo, nombre):
        if nombre < nodo.nombre:
            if nodo.izquierdo is None: nodo.izquierdo = Nodo(nombre)  # hueco
            else: self._insertar(nodo.izquierdo, nombre)
        elif nombre > nodo.nombre:
            if nodo.derecho is None: nodo.derecho = Nodo(nombre)    # hueco
            else: self._insertar(nodo.derecho, nombre)
        # igual → ignorar (no duplicados)

    def buscar(self, nombre):
        return self._buscar(self.raiz, nombre)

    def _buscar(self, nodo, nombre):
        if nodo is None: return False          # no existe
        if nombre == nodo.nombre: return True  # encontrado
        if nombre < nodo.nombre: return self._buscar(nodo.izquierdo, nombre)
        return self._buscar(nodo.derecho, nombre)

    def listar(self):  # inorden → siempre ordenado en BST
        lista = []
        self._inorden(self.raiz, lista)
        return lista

    def _inorden(self, nodo, lista):
        if nodo is None: return
        self._inorden(nodo.izquierdo, lista)
        lista.append(nodo.nombre)
        self._inorden(nodo.derecho, lista)

🔑 Prefijo _

Los métodos con _ son privados (auxiliares internos). No se llaman desde fuera de la clase. Separan la lógica pública de la recursiva.

🔑 Insertar vs Buscar

Ambos bajan por el árbol con la misma lógica (izq/der). La diferencia: insertar crea el nodo cuando llega a None; buscar devuelve False.

🔑 Listar en orden

Usar inorden en un BST da los valores ordenados. ¡Gratis! El árbol ya los tiene organizados por su estructura.

Resumen y Trucos de Examen

Comparativa de los tres recorridos

RecorridoOrden de visitaMnemotécnicoUso típico
Inorden izq → raíz → der "Sándwich: pan, relleno, pan" Valores ordenados en BST
Preorden raíz → izq → der "Yo primero, luego mis hijos" Copiar/serializar árbol
Postorden izq → der → raíz "Hijos primero, yo al final" Eliminar nodos, expresiones

Checklist para el examen ✅

1
Todo recorrido recursivo necesita caso base: if nodo is None: return. Sin él, el programa entra en bucle infinito.
2
En un BST: menores a la izquierda, mayores a la derecha. Inorden siempre da los valores ordenados.
3
La altura es recursiva: 1 + max(altura(izq), altura(der)). Un árbol vacío tiene altura 0.
4
Al insertar en BST: bajar comparando hasta encontrar None, ahí crear el nuevo Nodo.
5
Los métodos auxiliares recursivos suelen tener _ al inicio. Se llaman desde el método público que inicializa la llamada con la raíz.
💡 Regla de oro: si ves recursividad en árboles, pregúntate siempre:
1) ¿Cuál es el caso base? (normalmente nodo is None)
2) ¿Qué hago antes de llamar a los hijos? (preorden)
3) ¿Qué hago entre los hijos? (inorden)
4) ¿Qué hago después de los hijos? (postorden)
