Concepto fundamental

¿Qué es la recursividad?

Una función es recursiva cuando se llama a sí misma dentro de su propio cuerpo, resolviendo cada vez un problema más pequeño, hasta llegar a un caso tan simple que se resuelve directamente.

Concepto fundamental

Anatomía de una función recursiva

def mi_funcion(n):

    # ① CASO BASE ──────────────────────
    if n == 0:
        return 0

    # ② LLAMADA RECURSIVA ──────────────
    return n + mi_funcion(n - 1)
① Caso base ② Llamada recursiva

① Caso base

Condición if que devuelve un valor directamente, sin llamarse a sí misma. Es la condición de parada.

⚠️ Sin caso base → bucle infinito → error de pila (RecursionError)

② Llamada recursiva

Se llama a sí misma con un argumento más cercano al caso base (n−1, lista más corta, etc.). El resultado se usa para calcular la respuesta actual.

Ejemplo 1

Factorial — Código

def factorial(n):

    # CASO BASE
    if n == 0 or n == 1:
        return 1

    # LLAMADA RECURSIVA
    return n * factorial(n - 1)

Caso base

Si n es 0 o 1, devuelve 1 directamente. No hay más multiplicaciones posibles.

Llamada recursiva

Multiplica n por el factorial del número anterior: factorial(n-1). Cada llamada reduce n en 1.

Fórmula matemática

factorial(n) = n × (n−1) × … × 1
factorial(4) = 4 × 3 × 2 × 1 = 24

Ejemplo 1 — Traza

Factorial — Ejecución paso a paso

BAJADA → se apilan las llamadas

factorial(4) └─ 4 × factorial(3) └─ 3 × factorial(2) └─ 2 × factorial(1) └─ devuelve 1 ← CASO BASE

SUBIDA → se van resolviendo

2 × 1 = 2 3 × 2 = 6 4 × 6 = 24 ← resultado final

PILA DE LLAMADAS (memoria)

Tope de la pila → caso base
factorial(4) — espera
factorial(3) — espera
factorial(2) — espera
factorial(1) → devuelve 1

Prueba rápida:

factorial(0) = 1
factorial(1) = 1
factorial(4) = 24
factorial(5) = 120

Ejemplo 2

Suma de naturales — Código

def suma(n):

    # CASO BASE
    if n == 0:
        return 0

    # LLAMADA RECURSIVA
    return n + suma(n - 1)

Caso base

Si n == 0, no hay nada que sumar → devuelve 0.

Llamada recursiva

Suma n al resultado de sumar los anteriores: suma(n-1).

¿Por qué funciona?

suma(4) = 4 + suma(3)
= 4 + 3 + suma(2)
= 4 + 3 + 2 + suma(1)
= 4 + 3 + 2 + 1 + suma(0)
= 4 + 3 + 2 + 1 + 0
= 10

Ejemplo 2 — Traza

Suma de naturales — Ejecución paso a paso

BAJADA → acumulando llamadas

suma(4) └─ 4 + suma(3) └─ 3 + suma(2) └─ 2 + suma(1) └─ 1 + suma(0) └─ devuelve 0 ← CASO BASE

SUBIDA → calculando resultados

1 + 0 = 1 2 + 1 = 3 3 + 3 = 6 4 + 6 = 10 ← resultado final

Diferencia con Factorial

factorial(n)suma(n)
Operaciónmultiplicar ×sumar +
Caso basen == 0 o 1 → 1n == 0 → 0
Reducciónn - 1n - 1
suma(4)4×3×2×1 = 244+3+2+1 = 10

suma(0) = 0
suma(1) = 1
suma(4) = 10
suma(5) = 15

Ejemplo 3

Suma de lista — Código

def suma_lista(lista):

    # CASO BASE
    if len(lista) == 0:
        return 0

    # LLAMADA RECURSIVA
    return lista[0] + suma_lista(lista[1:])

Clave: slicing de listas

lista = [3, 1, 4, 2]
lista[0] = 3 ← primer elemento
lista[1:]= [1, 4, 2] ← el resto

Caso base

Lista vacía (len(lista) == 0) → devuelve 0. No hay elementos que sumar.

Llamada recursiva

Toma el primer elemento lista[0] y lo suma al resultado de suma_lista del resto lista[1:].

Cada llamada la lista se hace 1 elemento más corta, hasta quedar vacía.

Patrón con listas

En vez de reducir n-1, reducimos la lista con lista[1:]. El principio es el mismo: acercarse al caso base.

Ejemplo 3 — Traza

Suma de lista — Ejecución paso a paso

suma_lista([3, 1, 4, 2])

suma_lista([3, 1, 4, 2]) └─ 3 + suma_lista([1, 4, 2]) └─ 1 + suma_lista([4, 2]) └─ 4 + suma_lista([2]) └─ 2 + suma_lista([]) └─ devuelve 0 ← BASE

SUBIDA →

2 + 0 = 2 4 + 2 = 6 1 + 6 = 7 3 + 7 = 10 ← resultado final

¿Qué pasa con la lista?

llamada 1 [3, 1, 4, 2] → toma 3
llamada 2 [1, 4, 2] → toma 1
llamada 3 [4, 2] → toma 4
llamada 4 [2] → toma 2
llamada 5 [] CASO BASE → 0

suma_lista([]) = 0
suma_lista([5]) = 5
suma_lista([3,1,4,2]) = 10

Ejemplo 4

Máximo de lista — Código

def maximo(lista):

    # CASO BASE: lista de 1 elemento
    if len(lista) == 1:
        return lista[0]

    # Máximo del resto
    max_resto = maximo(lista[1:])

    # Comparar primero vs. máximo del resto
    if lista[0] > max_resto:
        return lista[0]
    else:
        return max_resto

Caso base

Lista de un solo elemento (len(lista) == 1). Ese único elemento ya es el máximo por definición.

⚠️ Aquí no usamos lista vacía porque comparar requiere al menos 1 elemento.

Idea principal

Guarda el máximo del resto en max_resto, luego compara con lista[0]. El ganador se devuelve hacia arriba.

Novedad respecto a suma

En vez de operar directamente (lista[0] + …), guardamos el resultado recursivo en una variable para poder compararlo.

Ejemplo 4 — Traza

Máximo de lista — Ejecución paso a paso

maximo([3, 7, 1, 5])

maximo([3, 7, 1, 5]) max_resto = maximo([7, 1, 5]) max_resto = maximo([1, 5]) max_resto = maximo([5]) └─ devuelve 5 ← BASE ¿1 > 5? No → devuelve 5 ¿7 > 5? Sí → devuelve 7 ¿3 > 7? No → devuelve 7 ← resultado

¿Qué pasa en cada comparación?

maximo([5]) = 5 caso base
lista[0]=1, max_resto=5 → ¿1>5? No → devuelve 5
lista[0]=7, max_resto=5 → ¿7>5? Sí → devuelve 7
lista[0]=3, max_resto=7 → ¿3>7? No → devuelve 7

maximo([5]) = 5
maximo([3,7,1,5]) = 7
maximo([4,2,9,6]) = 9

Resumen

Los 4 ejemplos de un vistazo

Función Caso base Llamada recursiva Ejemplo
factorial(n) n == 0 or n == 1 → 1 n * factorial(n-1) factorial(4) = 24
suma(n) n == 0 → 0 n + suma(n-1) suma(4) = 10
suma_lista(lista) len(lista) == 0 → 0 lista[0] + suma_lista(lista[1:]) suma_lista([3,1,4,2]) = 10
maximo(lista) len(lista) == 1 → lista[0] max_resto = maximo(lista[1:]) + comparación maximo([3,7,1,5]) = 7

Regla 1

El caso base siempre devuelve un valor directo, sin llamarse a sí mismo.

Regla 2

La llamada recursiva usa un argumento más pequeño o una lista más corta.

Regla 3

Sin caso base → recursión infinita → RecursionError. Siempre define el caso base primero.
