¿Qué es una clase?

Una clase es un molde. Define qué datos tiene algo y qué puede hacer.
Los objetos son los ejemplares reales creados a partir de ese molde.

📋

Clase

El plano o molde. Describe la forma de algo. Solo existe en el código.

⚔️

Objeto

Un ejemplar concreto creado desde la clase. Tiene sus propios datos reales.

🔧

Método

Una función dentro de la clase. Define lo que el objeto sabe hacer.

# Clase = molde ────────────────────────────────
class Personaje:
    def __init__(self, nombre, vida):
        self.nombre = nombre
        self.vida   = vida

# Objetos = ejemplares concretos ────────────────
heroe   = Personaje("Arthas", 100)   # objeto 1
enemigo = Personaje("Orco",   60)    # objeto 2

__init__ y self

__init__ es el constructor: se ejecuta automáticamente al crear el objeto.
self es la referencia al objeto recién creado — así cada objeto guarda sus propios datos.

class Personaje:
    def __init__(self, nombre, vida, ataque):
        self.nombre  = nombre   # dato del objeto
        self.vida    = vida     # dato del objeto
        self.ataque  = ataque   # dato del objeto

heroe   = Personaje("Arthas", 100, 15)
enemigo = Personaje("Orco",   60,  10)
objeto: heroe
nombre = "Arthas"
vida = 100
ataque = 15
objeto: enemigo
nombre = "Orco"
vida = 60
ataque = 10

Mismo molde → datos distintos. Los objetos son independientes entre sí.

Ejemplo 1 — Métodos del personaje

Los métodos son funciones dentro de la clase. Siempre reciben self para acceder a los datos del objeto.

def atacar(self, objetivo):
    # Resta vida al objetivo usando el ataque propio
    objetivo.vida -= self.ataque
    print(f"{self.nombre} ataca a {objetivo.nombre}"
          f" por {self.ataque} de daño.")

def mostrar_estado(self):
    # Imprime los datos actuales del personaje
    print(f"[{self.nombre}]  "
          f"Vida: {self.vida}  "
          f"Ataque: {self.ataque}")

↓ enemigo.vida -= heroe.ataque
enemigo.vida: 60 → 45
🛡 enemigo.atacar(heroe)
↓ heroe.vida -= enemigo.ataque
heroe.vida: 100 → 90

Ejemplo 1 — Código completo

class Personaje:
    def __init__(self, nombre, vida, ataque):
        self.nombre = nombre
        self.vida   = vida
        self.ataque = ataque

    def atacar(self, objetivo):
        objetivo.vida -= self.ataque
        print(f"{self.nombre} ataca a {objetivo.nombre} por {self.ataque} de daño.")

    def mostrar_estado(self):
        print(f"[{self.nombre}]  Vida: {self.vida}  Ataque: {self.ataque}")


heroe   = Personaje("Arthas", 100, 15)
enemigo = Personaje("Orco",    60, 10)

heroe.mostrar_estado()      # [Arthas]  Vida: 100  Ataque: 15
enemigo.mostrar_estado()    # [Orco]    Vida: 60   Ataque: 10

heroe.atacar(enemigo)       # Arthas ataca a Orco por 15 de daño.
enemigo.atacar(heroe)       # Orco ataca a Arthas por 10 de daño.

heroe.mostrar_estado()      # [Arthas]  Vida: 90   Ataque: 15
enemigo.mostrar_estado()    # [Orco]    Vida: 45   Ataque: 10

¿Qué es la Herencia?

Una clase hija puede heredar todo lo que tiene la clase padre.
No hay que repetir el código — y además puedes agregar cosas nuevas.

🧱 Personaje (padre)
nombre, vida, ataque
atacar(objetivo)
mostrar_estado()
hereda todo ↓
▼
🧙 Mago (hijo)
nombre, vida, ataque (heredados)
mana ← nuevo
atacar() (heredado)
lanzar_hechizo() ← nuevo
mostrar_estado() ← override

super() — llamar al padre

super() te da acceso al constructor (o cualquier método) de la clase padre.
Así no tienes que repetir self.nombre = nombre y el resto.

❌ Sin super — repetición innecesaria

class Mago(Personaje):
    def __init__(self, nombre, vida, ataque, mana):
        self.nombre = nombre  # repetido
        self.vida   = vida    # repetido
        self.ataque = ataque  # repetido
        self.mana   = mana

✅ Con super — limpio y correcto

class Mago(Personaje):
    def __init__(self, nombre, vida, ataque, mana):
        # Llama al __init__ del padre
        super().__init__(nombre, vida, ataque)
        # Solo agrega lo nuevo del Mago
        self.mana = mana
  • 📦super() reutiliza el trabajo ya hecho en el padre
  • 🔧Solo agregas lo nuevo en el hijo (mana)
  • 🧬Si el padre cambia, el hijo se actualiza automáticamente

Ejemplo 2 — método nuevo + override

El Mago tiene un método nuevo (lanzar_hechizo) y sobreescribe mostrar_estado para incluir el maná.

# Método NUEVO del Mago
def lanzar_hechizo(self, objetivo):
    costo   = 20
    daño    = self.ataque * 2   # doble de daño

    if self.mana >= costo:
        objetivo.vida -= daño
        self.mana    -= costo
        print(f"¡Hechizo! {daño} daño mágico.")
    else:
        print("Sin maná.")

# OVERRIDE: sobreescribe mostrar_estado del padre
def mostrar_estado(self):
    print(f"[{self.nombre}] Vida:{self.vida} "
          f"Maná:{self.mana}")

mana < 20 ❌
"Sin maná"
mana ≥ 20 ✅
vida -40
mana -20
🛡 guerrero.atacar(mago)
(método heredado)

Ejemplo 2 — Código completo

class Personaje:                               # PADRE
    def __init__(self, nombre, vida, ataque):
        self.nombre = nombre ; self.vida = vida ; self.ataque = ataque
    def atacar(self, obj): obj.vida -= self.ataque ; print(f"{self.nombre} ataca por {self.ataque}")
    def mostrar_estado(self): print(f"[{self.nombre}] Vida:{self.vida}")

class Mago(Personaje):                          # HIJO
    def __init__(self, nombre, vida, ataque, mana):
        super().__init__(nombre, vida, ataque)    # reutiliza al padre
        self.mana = mana
    def lanzar_hechizo(self, objetivo):
        if self.mana >= 20:
            objetivo.vida -= self.ataque * 2 ; self.mana -= 20
            print(f"¡Hechizo! {self.ataque*2} daño mágico. Maná: {self.mana}")
        else: print("Sin maná.")
    def mostrar_estado(self):                    # override
        print(f"[{self.nombre}] Vida:{self.vida} Maná:{self.mana}")

guerrero = Personaje("Guerrero", 120, 12)
mago     = Mago("Gandalf", 70, 20, 100)

guerrero.mostrar_estado()   # [Guerrero] Vida:120
mago.mostrar_estado()       # [Gandalf]  Vida:70 Maná:100
mago.lanzar_hechizo(guerrero) # ¡Hechizo! 40 daño. Maná: 80
guerrero.atacar(mago)         # Guerrero ataca por 12
guerrero.mostrar_estado()   # [Guerrero] Vida:80
mago.mostrar_estado()       # [Gandalf]  Vida:58 Maná:80

Clase

⚙️

Funciones del objeto. Modifican o consultan sus propios atributos.

Herencia + super()

La clase hija reutiliza al padre y agrega lo suyo. super().__init__(…)

Regla de oro: si dos clases comparten atributos/métodos, usa herencia. Si no, defínelas por separado. 🚀
